# Coding Architecture

## Overview
This document elaborates on the high-level coding architecture used in this repository, which is designed to promote modularity and maintainability.

## Layers
1. **Domain**: Core business logic.
2. **Presentation**: API controllers and views.
3. **Application**: Service layer.
4. **Infrastructure**: Data repositories, external APIs.

## Principles
- SOLID Principles: Writing clean, maintainable, and testable code.
- Clean Separation: Avoiding dependencies between layers.
